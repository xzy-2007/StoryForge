# StoryForge 项目规约（SPEC）

版本：v2.0

项目类型：AI4SE Final Project B —— 非 Harness 应用类项目

---

## 1. 问题陈述

### 1.1 要解决的问题

传统剧情设计依赖 Word 文档、思维导图或表格，在处理以下场景时效率低下：

- 多分支叙事结构难以可视化；
- 节点关系调整需要大量手动重绘；
- 缺乏对节点类型（对话、选择、事件等）的语义区分；
- 无法直观地查看故事流向。

### 1.2 目标用户

- 独立游戏开发者（需设计互动叙事分支）；
- 互动小说 / 视觉小说作者；
- 剧本创作者（需管理多线叙事）；
- 叙事设计爱好者。

### 1.3 为什么值得做

市面上缺乏轻量级、开源的专用剧情节点编辑器。现有方案要么是通用绘图工具（draw.io、Figma），要么是嵌入大型引擎（Unity、Unreal）的编辑器。StoryForge 填补了这个空白：一个专注于剧情节点编辑的轻量桌面工具。

## 2. 用户故事

1. **作为剧情作者**，我希望创建不同类型的剧情节点（对话、选择、事件等），以便区分不同叙事元素。
2. **作为剧情作者**，我希望能通过拖拽端口来连接节点，以便表示剧情的发展方向。
3. **作为剧情作者**，我希望拖拽节点来调整布局，以便整理故事结构。
4. **作为剧情作者**，我希望点击节点后在右侧面板编辑其标题、内容和类型，以便完善剧情信息。
5. **作为剧情作者**，我希望保存项目到本地存储并随时加载，以便分多次完成编辑。
6. **作为剧情作者**，我希望右键节点弹出快捷菜单来切换类型、复制或删除，以便快速操作。
7. **作为剧情作者**，我希望批量框选节点并整体移动，以便高效调整布局。
8. **作为剧情作者**，我希望导出故事为图片或 Markdown 文档，以便分享给他人。
9. **作为剧情作者**，我希望切换主题配色或自定义颜色，以便获得舒适的视觉体验。
10. **作为剧情作者**，我希望选择不同的节点形状样式，以便个性化编辑界面。

## 3. 功能规约

### 3.1 节点管理模块

| 功能 | 输入 | 行为 | 输出 | 边界条件 |
|---|---|---|---|---|
| 创建节点 | 无（随机类型） | 生成新节点并添加到画布 | 新节点出现在画布随机位置 | 无上限 |
| 删除节点 | 节点 ID | 删除节点及其所有关联连线 | 节点从画布消失 | 删除不存在的节点无影响 |
| 编辑节点 | 节点 ID + 字段值 | 更新节点的 title / content / type | 画布和属性面板同步更新 | type 必须是 NodeType 枚举值 |
| 复制节点 | 源节点对象 | 创建内容相同、位置偏移 50px 的新节点 | 新节点出现在画布 | 复制后自动选中新节点 |

### 3.2 连线管理模块

| 功能 | 输入 | 行为 | 输出 | 边界条件 |
|---|---|---|---|---|
| 创建连线 | source 端口 → target 端口 | 在两个节点间建立 Edge | 连接线出现在画布上 | 不允许自连 |
| 删除连线 | 选中连线后按 Delete | 从数据结构中移除 Edge | 连接线消失 | 删除不存在的连线无影响 |

### 3.3 可视化编辑器模块

| 功能 | 说明 |
|---|---|
| 节点拖拽 | 拖拽节点任意移动，实时更新位置 |
| 框选 | 在画布空白处拖拽可框选多个节点 |
| Shift 多选 | 按住 Shift 点击节点可追加选中 |
| 批量移动 | 选中多个节点后拖拽任意一个，整体移动 |
| 画布缩放 | 使用滚轮缩放，Controls 组件提供 +/- 和 fitView |
| 网格背景 | 显示网格辅助线 |

### 3.4 节点类型系统

| 类型 | 颜色 | 说明 |
|---|---|---|
| 💬 对话 (DIALOGUE) | `#4fc3f7` 蓝 | 角色对话节点 |
| 🔀 选择 (CHOICE) | `#ffb74d` 橙 | 玩家选择分支节点 |
| 📌 事件 (EVENT) | `#81c784` 绿 | 故事事件发生节点 |
| ❓ 条件 (CONDITION) | `#ce93d8` 紫 | 条件判断节点 |
| 🏁 结局 (ENDING) | `#ef5350` 红 | 故事结局节点 |
| 📝 备注 (NOTE) | `#90a4ae` 灰 | 作者备注节点 |

### 3.5 属性面板模块

- 位置：右侧边栏，宽度 220px，可折叠至 40px
- 选中节点时显示：标题、内容、类型（下拉选择器）、节点 ID
- 未选中节点时显示提示文字："← 点击节点编辑属性"
- 修改实时同步到画布

### 3.6 内容折叠功能

- 节点内容超过 100 字符时自动截断，显示"▼ 展开"按钮
- 点击展开后显示全文，按钮变为"▲ 收起"
- 短内容（≤100 字符）正常显示，无折叠按钮

### 3.7 主题系统

| 功能 | 说明 |
|---|---|
| 预设主题 | 深蓝、森林、日落、海洋、暗夜，共 5 套 |
| 自定义颜色 | 11 个颜色维度可独立自定义（背景、面板、边框、文字、各节点类型等） |
| 持久化 | 主题选择和自定义颜色保存到 localStorage |
| 影响范围 | 背景、工具栏、属性面板、输入框、节点卡片全部跟随主题 |

### 3.8 节点样式

| 样式 | 效果 |
|---|---|
| 圆角矩形 | 默认，`borderRadius: 8` |
| 胶囊形 | 超椭圆，`borderRadius: 24` |
| 卡片式 | 左侧彩色竖条强调 |

### 3.9 右键菜单模块

- 右键点击节点弹出上下文菜单
- 菜单项：切换类型（6 种）、复制节点、删除节点
- 点击菜单外部或按 Escape 关闭

### 3.10 导出模块

| 格式 | 实现方式 | 说明 |
|---|---|---|
| PNG | `html-to-image` 捕获 `.react-flow__viewport` | 3 倍像素比高清输出，仅节点区域 |
| Markdown | 手动生成结构化文档 | 包含节点列表、连接关系 |

### 3.11 保存/加载模块

- 格式：JSON，符合 `ProjectData` 接口
- 存储位置：`localStorage`（key: `storyforge-project`）
- 保存时弹出确认提示

### 3.12 设置面板

- 入口：工具栏"⚙️ 设置"按钮
- 内容：主题切换、自定义颜色、节点样式选择、使用说明
- 帮助入口：工具栏"❓ 帮助"按钮，自动滚动到使用说明区域

## 4. 非功能性需求

### 4.1 性能

- 支持 50+ 节点同时编辑不卡顿
- 节点拖拽、连线操作响应时间 < 100ms

### 4.2 安全（凭据威胁模型）

当前版本**不调用任何外部 API**（不含 LLM 调用、不含遥测上报），因此不存在 API Key 泄露风险。所有数据存储在用户本地的 `localStorage` 中，不上传任何服务器。

如未来加入 AI 辅助功能：

| 威胁 | 对策 |
|---|---|
| API Key 硬编码进源码 | 强制通过安全存储接口录入，禁止硬编码 |
| Key 被提交进 Git | `.gitignore` 排除所有包含 key 的文件，pre-commit hook 检查 |
| Key 在进程环境变量中泄露 | 使用操作系统钥匙串或加密文件存储，而非环境变量 |
| 日志中意外打印 Key | 日志过滤函数自动屏蔽疑似 Key 的字符串 |

### 4.3 可用性

- 界面语言：中文
- 所有操作均有视觉反馈（选中高亮、hover 效果、过渡动画）
- 工具栏右侧显示快捷操作提示

### 4.4 可维护性

- 模块职责清晰，组件按功能目录组织
- 核心逻辑有单元测试覆盖
- 构建和测试一键运行

### 数据流

```
用户操作 → React Flow (onNodesChange/onEdgesChange/onConnect)
         → App 状态管理 (useNodesState/useEdgesState)
         → 画布重新渲染
         → 属性面板同步更新
```

```
保存: nodes + edges → ProjectManager.saveProject() → JSON → localStorage
加载: localStorage → JSON → ProjectManager → setNodes/setEdges → 画布恢复
```

## 6. 数据模型

### 6.1 StoryNode

```typescript
interface StoryNode {
  id: string        // 自动生成，格式: "story-node-{counter}-{timestamp}"
  type: NodeType    // 枚举: DIALOGUE | CHOICE | EVENT | CONDITION | ENDING | NOTE
  title: string     // 节点标题
  content: string   // 节点内容（可空）
}
```

### 6.2 NodeType 枚举

```typescript
enum NodeType {
  DIALOGUE = 'DIALOGUE',
  CHOICE = 'CHOICE',
  EVENT = 'EVENT',
  CONDITION = 'CONDITION',
  ENDING = 'ENDING',
  NOTE = 'NOTE',
}
```

### 6.3 NodeShape 枚举

```typescript
enum NodeShape {
  ROUNDED = 'rounded',
  PILL = 'pill',
  CARD = 'card',
}
```

### 6.4 ProjectData

```typescript
interface ProjectData {
  version: string    // 格式版本，当前 "1.0"
  name: string       // 项目名称
  nodes: Node[]      // React Flow Node 数组
  edges: Edge[]      // React Flow Edge 数组
}
```

## 7. 凭据与分发设计

### 7.1 凭据安全

当前版本**无 API 调用需求**，不存在凭据管理问题。架构中预留了 `ProjectManager` 服务层，未来如需接入 AI 功能，将通过以下安全方式管理 Key：

1. **录入**：设置面板中提供 Key 输入框，输入时掩码显示
2. **存储**：使用 `localStorage` 加密存储（基础版）/ 操作系统钥匙串（进阶版）
3. **查看/更新/清除**：设置面板中可查看状态（不回显明文）、更新、清除
4. **防护**：`.gitignore` 排除所有可能包含 Key 的文件；pre-commit hook 检查 Key 泄露

### 7.2 分发

本项目提供两种分发形态：

**A. 容器镜像（Docker）**

```bash
docker build -t storyforge .
docker run -p 5173:5173 storyforge
```

适用于 CI/CD 和服务器端预览。构建产物为静态文件，通过 `electron-vite preview` 提供 HTTP 服务。

**B. Electron Builder 原生二进制**

通过 `electron-builder` 打包为 Windows / macOS / Linux 原生桌面应用：

```bash
npm run build:electron
```

产物在 `dist/` 目录下，包含可执行安装程序。

## 8. 技术选型与理由

| 技术 | 选型理由 |
|---|---|
| **TypeScript** | 类型安全，适合中等规模项目维护 |
| **React 18** | 组件化 UI，生态成熟 |
| **Electron** | 跨平台桌面应用，方便交付 |
| **Vite (electron-vite)** | 极速 HMR，统一管理主进程/渲染进程构建 |
| **React Flow** | 成熟的节点图编辑器库，提供拖拽、连线、缩放等核心能力，避免从零实现图形编辑器 |
| **Vitest** | 与 Vite 深度集成，运行速度快 |
| **html-to-image** | 将 DOM 节点转为 PNG，用于导出功能 |
| **Docker** | 标准化分发，确保环境一致性 |

## 10. 风险与未决问题

| 风险 | 影响 | 缓解措施 |
|---|---|---|
| React Flow 版本升级破坏 API | 编辑器无法工作 | 锁定 `reactflow@^11.11.4` 版本 |
| Electron 打包体积过大 | 分发困难 | 使用 `electron-builder` 压缩，按平台分别打包 |
| localStorage 容量限制（5MB） | 大项目无法保存 | 未来可切换为文件系统保存 |
| 无 Undo/Redo | 用户误操作无法撤销 | 已知限制，标注在 README 中 |
| 无多语言支持 | 仅中文界面 | 已知限制，标注在 README 中 |
