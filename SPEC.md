# 1. 项目简介

## 1.1 项目名称

StoryForge

## 1.2 项目定位

StoryForge 是一个面向剧情创作者的可视化剧情编辑工具。

用户可以通过类似 Unity 编辑器的节点操作方式，将剧情拆分为多个节点，并通过连接关系表示剧情发展。

## 1.3 解决的问题

传统剧情设计通常依赖：

- Word 文档；
- 思维导图；
- 表格；

这些方式在处理以下场景时效率较低：

- 多分支剧情；
- 剧情关系查看；
- 节点调整。

StoryForge 希望提供一个更加直观的剧情结构编辑方式。

# 4. 数据模型

## 4.1 剧情节点 Node

一个剧情节点表示剧情中的一个事件。

### 字段

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 节点唯一编号 |
| type | NodeType | 节点类型 |
| title | string | 节点标题 |
| description | string | 剧情描述 |
| position | Position | 节点在画布中的位置 |

### 示例

```json
{
  "id": "node1",
  "type": "DIALOGUE",
  "title": "第一次见面",
  "description": "主角遇到了伙伴",
  "position": {
    "x": 100,
    "y": 200
  }
}
```

## 4.2 节点类型 NodeType

节点类型使用字符串表示。

| 类型 | 说明 |
|------|------|
| EVENT | 事件节点 |
| DIALOGUE | 对话节点 |
| CHOICE | 选择节点 |
| END | 结束节点 |

程序中：

```typescript
enum NodeType {
  EVENT = "EVENT",
  DIALOGUE = "DIALOGUE",
  CHOICE = "CHOICE",
  END = "END"
}
```

使用字符串形式原因：

- 保存到 JSON 后容易阅读；
- 用户可以直接理解项目文件。

## 4.3 Position

表示节点在画布中的位置。

| 字段 | 类型 |
|------|------|
| x | number |
| y | number |

## 4.4 Edge

Edge 表示两个剧情节点之间的连接。

### 字段

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 连接编号 |
| source | string | 起始节点 id |
| target | string | 目标节点 id |
| label | string | 连接条件说明 |

### 示例

```json
{
  "id": "edge1",
  "source": "node1",
  "target": "node2",
  "label": "选择接受"
}
```

## 4.5 剧情项目 Project

Project 表示一个完整的剧情设计项目，是保存与加载的基本单位。

### 字段

| 字段 | 类型 | 说明 |
|------|------|------|
| version | string | 项目文件格式版本 |
| name | string | 项目名称 |
| nodes | Node[] | 项目内全部节点 |
| edges | Edge[] | 项目内全部连接 |

### 示例

```json
{
  "version": "1.0",
  "name": "我的剧情项目",
  "nodes": [],
  "edges": []
}
```

# 5. 数据创建方式

为了保证创建的数据格式正确，程序可以提供以下两个创建函数：

- `createNode()`
- `createEdge()`

作用：

- 自动生成 id；
- 设置默认值；
- 返回符合数据结构的对象。

例如：

```typescript
createNode({
  type: "DIALOGUE",
  title: "对话",
  description: "剧情内容"
})
```

返回：

```json
{
  "id": "自动生成",
  "type": "DIALOGUE",
  "title": "对话",
  "description": "剧情内容",
  "position": {
    "x": 0,
    "y": 0
  }
}
```

# 6. 功能详细说明

## 6.1 剧情节点管理

### 创建节点

输入：节点类型、标题、描述。

结果：生成新的剧情节点。

### 删除节点

删除节点时，同时删除所有与该节点相关的连接。

### 修改节点

支持修改：

- 标题；
- 描述；
- 类型。

## 6.2 剧情连接管理

### 创建连接

采用端口拖拽方式创建连接。

用户操作：

1. 鼠标按住节点输出端口；
2. 拖动至目标节点输入端口；
3. 松开鼠标；
4. 系统创建 Edge。

要求：

- 起点必须存在；
- 终点必须存在；
- 不允许节点连接自身。

连接限制：

- 一个输出端口允许多个连接；
- 一个输入端口默认只允许一个连接。

### 删除连接

用户操作：

1. 点击连接线；
2. 连接线进入选中状态；
3. 按 Delete 键删除。

删除后 Edge 从数据结构中移除，UI 中对应连接消失。

### 修改连接信息

支持修改连接标签（label），用于表示剧情选择条件。

示例：

```
          是
选择节点 --------> 结局A

          否
选择节点 --------> 结局B
```

## 6.3 可视化编辑器

整体布局（与 PLAN Task 7 对齐）：

```
--------------------------------
菜单栏
--------------------------------
资源区 | Canvas（剧情画布） | Inspector（属性面板）
--------------------------------
状态栏
```

各区域职责：

| 区域 | 职责 |
|------|------|
| 菜单栏 | 文件操作入口（新建、保存、加载等） |
| 资源区 | 预留扩展区域，MVP 可显示项目基本信息 |
| Canvas | 节点显示、拖动、连线创建与删除 |
| Inspector | 显示并编辑当前选中节点的属性 |
| 状态栏 | 显示基础提示信息 |

支持：

- 节点显示
- 节点拖动
- 连线显示
- 删除连接

#### 节点显示

显示：

- 节点标题；
- 节点类型。

#### 节点拖动

用户可以拖动节点，移动后保存新的坐标。

#### 连线显示

显示节点之间的连接关系。

#### 删除连接

点击连接线后可以删除。

## 6.4 属性编辑器

用户选择节点后，右侧显示属性。

可以修改：

- 标题；
- 描述；
- 类型。

修改后数据同步更新。

## 6.5 项目保存

### 保存项目

保存格式：JSON，数据结构为 Project（见 4.5）。

### 加载项目

读取 JSON 后恢复：

- 节点；
- 连接；
- 位置信息。
